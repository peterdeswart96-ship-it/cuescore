// 01 — ARCHITECTUUR

Hoe het systeem werkt

Een Azure Function App pollt elke 2 minuten de Cuescore API. Zodra een wedstrijd een tafelnummer krijgt stuurt het script via Ntfy.sh een pushmelding naar de betreffende spelers. De state wordt persistent opgeslagen in Azure Blob Storage zodat herstarts geen dubbele meldingen veroorzaken.

// 04 — ONTWIKKELING

Van idee naar productie

Negen iteratieve fases — inclusief twee mislukte pogingen die tot betere oplossingen leidden.

FASE 01
API Verkenning
Cuescore beta API gedocumenteerd via live toernooidata. Veldnamen ontdekt: playerA/B, matchstatus, table.name, scoreA/B.
CUESCORE APIPOWERSHELL
FASE 02 — MISLUKT
WhatsApp via CallMeBot
API-key activatie werkte na twee weken nog steeds niet. Besloten over te stappen naar een betere oplossing.
CALLMEBOTWHATSAPP
FASE 03
Ntfy integratie
Overstap naar Ntfy.sh — geen account, geen activatie. Pure HTTP POST naar een topic. Per-speler topics op basis van Cuescore player-ID.
NTFY.SHHTTP POSTTOPICS
FASE 04
GitHub Actions
Python script + scheduled workflow. Vertragingen tot 30 minuten tijdens piekuren — onbetrouwbaar voor real-time meldingen.
PYTHONGITHUB ACTIONS
FASE 05 — MISLUKT
Azure Flex Consumption
Flex Consumption plan (Linux/Python) blokkeerde uitgaand internetverkeer zonder VNet + NAT Gateway. Te complex en te duur.
FLEX CONSUMPTIONVNET BLOKKADE
FASE 06
Azure Function App — PowerShell
Consumption Windows met PowerShell 7.4. Volledige internettoegang, gegarandeerde timer elke 2 minuten. Stabiel en goedkoop.
POWERSHELL 7.4CONSUMPTION WINDOWSTIMER TRIGGER
FASE 07
State & auto-update
State opgeslagen in Azure Blob Storage — persistent over herstarts. Spelerslijst automatisch bijgewerkt via GitHub API na elk toernooi.
BLOB STORAGEGITHUB APIAUTO-UPDATE
FASE 08
Vier notificatietypes + webpagina
Uitbreiding naar vier kanalen: eigen wedstrijden, speler volgen, toernooi resultaten en wedstrijd leiding. Spelerspagina live op GitHub Pages.
